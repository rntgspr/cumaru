# Attachments

Retained source files that should outlive inbox cleanup. Use this pillar for
PDFs, exported pages, images, audio files, or other evidence that remains useful
after a capture is distilled.

## Rules

- **One directory per retained source set.** Use `attachments/<slug>/index.md`
  and store related files beside that index.
- **Retain intentionally.** Do not move every raw capture here; most inbox
  entries should be deleted after distillation.
- **Describe why it remains.** `retained-for` should point to the memory or
  draft that still needs the file.

## When to use

- A PDF, image, audio file, or export needs to remain available.
- The memory cannot be responsibly maintained from URLs/search queries alone.

## When NOT to use

- Temporary capture files -> keep in `inbox/` until processed.
- Textual, durable knowledge -> `memories/`.
